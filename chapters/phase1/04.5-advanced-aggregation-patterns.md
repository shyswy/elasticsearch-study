## 4.5단계: 고급 집계 패턴

> **학습 목표**: 대량 버킷 페이지네이션, 그룹별 상위 문서 추출, 조건부 버킷 필터링 등
> 실전에서 자주 쓰이는 고급 집계 패턴을 익힌다.

### 4.5-1. top_hits — 그룹별 상위 N개 문서 반환

`terms` 집계로 그룹핑한 뒤, 각 그룹의 실제 문서를 가져오는 패턴:

```json
// "issue_type별 top 3 최신 이슈 문서"
GET /issue-history/_search
{
  "size": 0,
  "aggs": {
    "by_issue_type": {
      "terms": { "field": "issue_type", "size": 10 },
      "aggs": {
        "top_issues": {
          "top_hits": {
            "size": 3,
            "sort": [{ "created_at": { "order": "desc" } }],
            "_source": ["issue_id", "title", "status", "assignee", "created_at"]
          }
        }
      }
    }
  }
}
```

**collapse vs top_hits 비교**:

| 기능 | collapse + inner_hits | terms + top_hits |
|------|----------------------|-----------------|
| 위치 | 검색 결과 레벨 | 집계 레벨 |
| 다른 집계와 조합 | ✗ (집계와 동시 사용 제한) | ✓ (다른 metric과 함께 가능) |
| 그룹 수 제어 | size로 전체 결과 수 제한 | terms size로 그룹 수 제한 |
| 추가 통계 | ✗ | ✓ (같은 레벨에 avg, sum 등 추가 가능) |

**top_hits + 통계 조합**:
```json
// "issue_type별 top 3 문서 + 평균 해결 시간 + 총 건수"
{
  "size": 0,
  "aggs": {
    "by_issue_type": {
      "terms": { "field": "issue_type", "size": 10 },
      "aggs": {
        "top_issues": {
          "top_hits": {
            "size": 3,
            "sort": [{ "created_at": { "order": "desc" } }],
            "_source": ["issue_id", "title", "created_at"]
          }
        },
        "avg_resolution": {
          "avg": { "field": "resolution_time_minutes" }
        },
        "open_count": {
          "filter": { "term": { "status": "open" } }
        }
      }
    }
  }
}
```

### 4.5-2. filters aggregation — 다중 조건 동시 집계

여러 조건을 한 번의 쿼리로 동시에 집계:

```json
// "status별 건수를 한 번에 (named filters)"
GET /issue-history/_search
{
  "size": 0,
  "aggs": {
    "status_breakdown": {
      "filters": {
        "filters": {
          "open": { "term": { "status": "open" } },
          "in_progress": { "term": { "status": "in_progress" } },
          "resolved": { "term": { "status": "resolved" } },
          "critical_open": {
            "bool": {
              "filter": [
                { "term": { "status": "open" } },
                { "term": { "priority": "critical" } }
              ]
            }
          }
        }
      }
    }
  }
}
```

**vs terms aggregation**:
- `terms`: 필드의 모든 고유값을 자동으로 그룹핑
- `filters`: 직접 정의한 조건별로 그룹핑 (복합 조건 가능, 이름 지정 가능)

### 4.5-3. composite aggregation — 대량 버킷 페이지네이션

`terms`는 기본 10개(최대 10,000개) 버킷만 반환. 전체 버킷을 순회하려면 `composite` 사용:

```json
// 첫 페이지
GET /issue-history/_search
{
  "size": 0,
  "aggs": {
    "all_combinations": {
      "composite": {
        "size": 100,
        "sources": [
          { "issue_type": { "terms": { "field": "issue_type" } } },
          { "status": { "terms": { "field": "status" } } }
        ]
      },
      "aggs": {
        "avg_resolution": {
          "avg": { "field": "resolution_time_minutes" }
        }
      }
    }
  }
}

// 다음 페이지 — 이전 응답의 after_key 사용
GET /issue-history/_search
{
  "size": 0,
  "aggs": {
    "all_combinations": {
      "composite": {
        "size": 100,
        "sources": [
          { "issue_type": { "terms": { "field": "issue_type" } } },
          { "status": { "terms": { "field": "status" } } }
        ],
        "after": { "issue_type": "incident", "status": "resolved" }
      }
    }
  }
}
```

**composite의 장점**:
- 모든 버킷 조합을 빠짐없이 순회 가능
- 메모리 효율적 (한 번에 일부만 로드)
- 다차원 그룹핑 (여러 필드 조합)

**composite에서 date_histogram 사용**:
```json
{
  "composite": {
    "size": 50,
    "sources": [
      { "date": { "date_histogram": { "field": "created_at", "calendar_interval": "day" } } },
      { "type": { "terms": { "field": "issue_type" } } }
    ]
  }
}
// → 날짜 × issue_type 모든 조합을 페이지네이션
```

### 4.5-4. bucket_sort — 집계 결과 정렬 & 페이지네이션

```json
// "서비스별 에러 수 → 에러 많은 순 상위 5개만"
GET /logs-*/_search
{
  "size": 0,
  "aggs": {
    "by_service": {
      "terms": { "field": "service", "size": 50 },
      "aggs": {
        "error_count": {
          "filter": { "term": { "level": "ERROR" } }
        },
        "sort_by_errors": {
          "bucket_sort": {
            "sort": [{ "error_count._count": { "order": "desc" } }],
            "size": 5
          }
        }
      }
    }
  }
}
```

**bucket_sort vs terms의 order**:
- `terms`의 `order`는 해당 집계 내에서만 정렬
- `bucket_sort`는 sub-aggregation 결과 기준으로 정렬 가능 + 페이지네이션(`from`) 지원

### 4.5-5. bucket_selector — 조건부 버킷 필터링

특정 조건을 만족하는 버킷만 남기기:

```json
// "평균 해결 시간이 120분 이상인 팀만"
GET /issue-history/_search
{
  "size": 0,
  "aggs": {
    "by_team": {
      "terms": { "field": "team", "size": 50 },
      "aggs": {
        "avg_resolution": {
          "avg": { "field": "resolution_time_minutes" }
        },
        "slow_teams_only": {
          "bucket_selector": {
            "buckets_path": {
              "avg_time": "avg_resolution"
            },
            "script": "params.avg_time > 120"
          }
        }
      }
    }
  }
}
```

**"이슈 10건 이상인 타입만"**:
```json
"having_enough_issues": {
  "bucket_selector": {
    "buckets_path": {
      "count": "_count"
    },
    "script": "params.count >= 10"
  }
}
```

### 4.5-6. Pipeline Aggregation 추가 패턴

**cumulative_sum — 누적합**:
```json
// "일별 이슈 누적 건수"
{
  "aggs": {
    "daily": {
      "date_histogram": { "field": "created_at", "calendar_interval": "day" },
      "aggs": {
        "daily_count": { "value_count": { "field": "issue_id" } },
        "cumulative_issues": {
          "cumulative_sum": { "buckets_path": "daily_count" }
        }
      }
    }
  }
}
```

**derivative — 변화율 (전일 대비 증감)**:
```json
// "일별 에러 수의 전일 대비 변화량"
{
  "aggs": {
    "daily": {
      "date_histogram": { "field": "@timestamp", "calendar_interval": "day" },
      "aggs": {
        "error_count": {
          "filter": { "term": { "level": "ERROR" } }
        },
        "error_change": {
          "derivative": { "buckets_path": "error_count._count" }
        }
      }
    }
  }
}
// 결과: 7/14: +20 (전일 대비 20건 증가), 7/15: -5 (5건 감소)
```

**percentiles — 백분위수**:
```json
// "응답 시간의 p50, p90, p99"
{
  "aggs": {
    "response_percentiles": {
      "percentiles": {
        "field": "response_time_ms",
        "percents": [50, 90, 95, 99]
      }
    }
  }
}
```

### 4.5-7. 실전 조합 예시

**"최근 30일, issue_type별 top 5 + 각 타입의 open 건수 + 평균 해결 시간"**:
```json
GET /issue-history/_search
{
  "size": 0,
  "query": {
    "range": { "created_at": { "gte": "now-30d" } }
  },
  "aggs": {
    "top_issue_types": {
      "terms": { "field": "issue_type", "size": 5 },
      "aggs": {
        "open_count": {
          "filter": { "term": { "status": "open" } }
        },
        "avg_resolution": {
          "avg": { "field": "resolution_time_minutes" }
        },
        "recent_examples": {
          "top_hits": {
            "size": 2,
            "sort": [{ "created_at": { "order": "desc" } }],
            "_source": ["issue_id", "title", "status"]
          }
        }
      }
    }
  }
}
```

**"주간 에러 추이 + 전주 대비 변화율 + 누적"**:
```json
GET /logs-*/_search
{
  "size": 0,
  "query": {
    "bool": {
      "filter": [
        { "term": { "level": "ERROR" } },
        { "range": { "@timestamp": { "gte": "now-90d" } } }
      ]
    }
  },
  "aggs": {
    "weekly": {
      "date_histogram": { "field": "@timestamp", "calendar_interval": "week" },
      "aggs": {
        "weekly_change": {
          "derivative": { "buckets_path": "_count" }
        },
        "cumulative_total": {
          "cumulative_sum": { "buckets_path": "_count" }
        }
      }
    }
  }
}
```

### 4.5-8. 학습 체크

- [ ] `top_hits`로 그룹별 상위 문서를 추출할 수 있다
- [ ] `collapse + inner_hits`와 `terms + top_hits`의 차이를 설명할 수 있다
- [ ] `filters` aggregation으로 다중 조건 동시 집계를 작성할 수 있다
- [ ] `composite` aggregation으로 대량 버킷을 페이지네이션할 수 있다
- [ ] `bucket_sort`로 집계 결과를 sub-agg 기준으로 정렬할 수 있다
- [ ] `bucket_selector`로 조건을 만족하는 버킷만 필터링할 수 있다
- [ ] `cumulative_sum`과 `derivative`의 차이를 설명할 수 있다
- [ ] 실전 시나리오에서 적절한 집계 조합을 선택할 수 있다
